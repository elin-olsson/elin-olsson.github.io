ssh-tls-auditor — Technical Report

A Python-based CLI tool that audits the security configuration of networked services. By connecting to SSH, TLS, HTTP, SMTP, FTP, and RDP endpoints, it evaluates cryptographic strength, certificate validity, protocol weaknesses, and security headers — then assigns an overall grade based on industry best practices.

FileContent

Core Functionality

1. Protocol Coverage

The auditor runs a dedicated check module for each protocol:

ProtocolWhat is checked
SSHKey exchange algorithms, ciphers, MACs, host key types, root login, password auth, legacy protocol support, banner fingerprinting
TLS/HTTPSProtocol versions (SSLv3–TLS 1.3), cipher suites, certificate validity, expiry, CN/SAN matching, OCSP status, signature algorithm
HTTPSecurity headers (HSTS, CSP, X-Frame-Options, etc.), HSTS preload status
SMTPSTARTTLS support, open relay test, DMARC, SPF, DKIM, DNS CAA/DNSSEC/TLSA
FTPAnonymous login, plaintext credential exposure
RDPNLA enforcement, deprecated RDP security layer detection

2. Grading

Each target receives an overall grade based on the number and severity of findings. Critical failures drive the grade down faster than warnings. The grading scale runs A (no failures) through F (four or more critical findings).

3. IDS-Aware Scanning

The --conn-delay SECONDS flag inserts a configurable delay between every connection attempt. This softens the scan profile against rate-limiting defences and IDS systems without skipping any checks.

Technical Challenges & Solutions

  • SSH algorithm enumeration without authentication: The auditor captures the server's KEXINIT packet during the key exchange handshake to extract supported algorithms — without completing the handshake or requiring credentials.
  • TLS cipher negotiation: Supported cipher suites are identified by probing the server repeatedly with different cipher lists, since a single TLS handshake only reveals what was negotiated, not the full set the server accepts.
  • IDS triggering: During testing against Suricata in IPS mode, rapid connection attempts triggered ET SCAN rules and caused active blocks. The --conn-delay flag was added as a result, providing a per-connection sleep that reduces burst patterns.
  • External target detection: The tool warns when a target resolves outside RFC1918/loopback ranges, preventing accidental scanning of public infrastructure.
