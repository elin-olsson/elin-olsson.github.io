chain-watch — Technical Report

A Python-based CLI security tool designed to correlate security logs on Linux systems. By analyzing auth.log, firewall logs, and auditd events in parallel, it identifies attack chains and patterns that single-log monitors might miss.

Core Functionality

1. Multi-Source Parsers

The tool handles four distinct log sources:

  • Auth/Secure: Tracks successful/failed SSH logins and sudo execution.
  • Firewall: Detects UFW, Firewalld, Iptables, and Nftables logs.
  • Auditd: Correlates syscalls and execution arguments (execve) with user identity.
  • Systemd Journal: Reads SSH, sudo, and kernel events via journalctl — merged with any file-based sources using --journal.

2. Correlation Logic

Events are grouped by source IP and analyzed within a sliding time window (default 600s). Key detection patterns include:

Detection ChainSeverityTrigger
Brute ForceMedium≥ 5 failed logins within window
Brute then LoginCriticalBrute force cluster followed by success
Scan then LoginHighFirewall block followed by login attempt
Lateral MovementHigh/CriticalLogin followed by sensitive command execution or privilege escalation
Credential StuffingHigh≥ 5 distinct usernames attempted from same IP within window

Technical Challenges & Solutions

During development, several complex issues were addressed:

  • Auditd Hex Decoding: Auditd often encodes arguments in hex. A custom decoder was implemented to normalize these into readable strings.
  • Time Normalization: Different logs use different timestamp formats. The tool normalizes everything to a standard UTC datetime object for precise correlation.
  • User-IP Mapping: Auditd doesn't log IPs for syscalls. The tool maps these back to the source IP by tracking the user's last successful SSH session.
