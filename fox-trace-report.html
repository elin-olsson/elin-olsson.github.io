fox-trace — Technical Report

A Python-based CLI tool that maps SSH trust relationships across a Linux system. By parsing ~/.ssh/known_hosts, authorized_keys, private keys, and running SSH agents, it constructs a Shadow Map — a graph of which hosts and identities can reach each other without a password.

Core Functionality

1. SSH Artefact Collection

The harvester scans the user's ~/.ssh/ directory and collects:

  • Private keys — type (RSA/Ed25519/ECDSA), encryption status, age, permissions
  • Known hosts — all trusted remote hosts, including hashed entries
  • Authorized keys — inbound trust: who can log in as the current user
  • SSH config — HostName/IdentityFile aliases that expand the real attack surface
  • SSH agents — live agent sockets in /tmp/ssh-* that could be hijacked

2. Trust Graph & Blast Radius

All collected hosts and identities are assembled into a trust graph. The blast radius metric counts how many distinct hosts are reachable from the current system using only the keys found on disk. This is the number an attacker gains access to if this machine is compromised.

3. Risk Scoring & Alerts

Each finding is classified with a risk score. Example alert conditions:

AlertRiskCondition
Unencrypted private keyCriticalKey file has no passphrase
Wrong key permissionsHighPrivate key is world- or group-readable
Stale key (>180 days)HighKey has not been rotated
Live SSH agentMediumAgent socket found in /tmp — hijackable
RSA key <4096 bitsMediumKey size below recommended threshold

4. GitHub Key Verification

If a GitHub username is supplied with --github USERNAME, the tool fetches the user's public keys from the GitHub API and cross-references them against local authorized_keys. This reveals whether any GitHub identity has been granted inbound SSH access.

Output Formats

fox-trace produces three output modes:

  • Terminal — colour-coded summary with risk score, blast radius, and alerts
  • JSON — machine-readable full dump of all artefacts and findings
  • HTML — interactive Shadow Map visualisation using D3.js force-directed graph (nodes = hosts/keys, edges = trust relationships)

Technical Challenges & Solutions

  • RSA key bit length: Public keys in authorized_keys are base64-encoded blobs. A custom parser decodes the wire format (type field + modulus) to extract the actual bit count without shelling out to ssh-keygen.
  • Hashed known_hosts: OpenSSH optionally hashes hostnames with HMAC-SHA1. The tool detects hashed entries and marks them as opaque rather than skipping or misreporting them.
  • SSH config aliases: A Host block can map a short alias to a real hostname and a specific key. The parser resolves these so the trust graph reflects actual connectivity, not just what known_hosts contains.
